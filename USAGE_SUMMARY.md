### README 编写规范

所有项目 README 必须包含：

1. **是什么？**
   - 项目介绍
   - 核心功能
   - 适用场景

2. **为什么？**
   - 解决的问题
   - 优势特点
   - 应用价值

3. **怎么用？**
   - 快速开始
   - 详细使用
   - 配置说明

### 关键原则

- ✅ 中文优先（项目介绍）
- ✅ 实用导向（应用层面）
- ✅ 避免过度类比（只在必要时）
- ✅ 代码示例（可运行的示例）
